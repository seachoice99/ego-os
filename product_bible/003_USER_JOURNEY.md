# User Journey

## Базовый сценарий

1. Пользователь открывает Ego OS.
2. Видит Dashboard: активные задачи, сотрудники, расходы, блокеры, результаты.
3. Создаёт задачу естественным языком.
4. Orchestrator анализирует намерение, сложность, риски и нужные компетенции.
5. Система формирует план.
6. Назначает существующих сотрудников или создаёт недостающие роли.
7. Пользователь видит, кто приступил к работе.
8. В реальном времени обновляются статусы и отчёты.
9. После выполнения задача проходит QA.
10. Пользователь получает результат, отчёт, стоимость и сохранённые артефакты.
11. Полезные решения попадают в Memory.

## Ключевое ощущение

Пользователь должен чувствовать: «у меня есть цифровая команда, которая работает прозрачно и управляемо».
