# Employee Update Workflow

Employees are updated like software modules.

## Reasons to update

- better model available;
- new tool integration;
- recurring quality issue;
- user feedback;
- cost optimization;
- new responsibilities.

## Update format

- employee id;
- previous version;
- new version;
- change summary;
- reason;
- migration notes;
- rollback plan.

## Rule

Old task reports must continue referencing the employee version that actually performed the work.
