Each row's "Canonical decision" column reflects the Owner decision already recorded in the directing prompt for this pass (see the five new ADRs this audit feeds, `memory/decisions/ADR-0013` through `ADR-0017`) — this document does not itself decide anything; it maps evidence to already-made decisions and names what still needs to change to match them.

Methodology: three parallel read-only discovery passes (documentation/ADR layer, Python runtime layer, employees/skills/automation-runner layer), each citing exact file:line evidence. No file outside this one was modified to produce this audit.

## Legend

- **Type**: `doc-vs-doc` (two normative documents disagree with each other), `doc-vs-code` (a document's claim doesn't match what the running code does), `code-gap` (no contradiction, but a described capability has zero implementation), `security-gap` (a stated security invariant is not actually enforced in code).
