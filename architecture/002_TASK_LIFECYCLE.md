# Task Lifecycle

1. Intake
   - user submits goal or request.

2. Clarification Check
   - system decides whether it has enough information.
   - if not, asks only critical questions.

3. Planning
   - Orchestrator creates task plan.
   - estimates complexity, budget, required roles and risks.

4. Staffing
   - existing employees are selected.
   - missing roles are created or proposed.

5. Execution
   - employees perform subtasks.
   - all significant actions create log events.

6. QA
   - result is checked against request, project context and quality rules.

7. Delivery
   - final result is shown to user.
   - report includes who did what, cost, tokens, tools and files.

8. Memory Update
   - decisions, reusable findings and artifacts are stored.
