# Reporting and Logs

## Mandatory task report

Every task must produce a report with:

- task summary;
- employees involved;
- timeline;
- status changes;
- decisions made;
- models and tools used;
- token usage;
- money spent;
- generated artifacts;
- unresolved questions;
- memory updates.

## Event log format

Each work event should include:

- timestamp;
- employee_id;
- action_type;
- human_readable_summary;
- input_reference;
- output_reference;
- usage;
- cost;
- status.

## User-facing rule

Logs should not expose raw hidden reasoning. They should expose operational reasoning: what was done, why it was done, and what changed.
